# SunnyFilms 部署指南

## 方案一：前端 Cloudflare Pages + 后端 Vercel（推荐）

### 前置准备

1. 注册 GitHub 账号：https://github.com
2. 注册 Cloudflare 账号：https://dash.cloudflare.com/sign-up
3. 注册 Vercel 账号：https://vercel.com/signup

### 步骤 1：上传代码到 GitHub

1. 在 GitHub 创建一个新仓库
2. 将项目代码推送到 GitHub 仓库

### 步骤 2：部署后端到 Vercel

1. 访问 https://vercel.com/new
2. 导入你的 GitHub 仓库
3. 点击 "Deploy"
4. 等待部署完成，你会获得一个类似 `https://sunnyfilms.vercel.app` 的地址

**注意**：Vercel 是无服务器环境，本地文件存储（data/ 目录）不会持久化。需要持久化数据的话，建议使用：
- Vercel KV（键值存储）
- Vercel Postgres（数据库）

### 步骤 3：部署前端到 Cloudflare Pages

1. 访问 https://dash.cloudflare.com
2. 进入 "Workers & Pages" → "Create application" → "Pages"
3. 选择 "Connect to Git"
4. 选择你的 GitHub 仓库
5. 配置构建设置：
   - **Project name**: sunnyfilms
   - **Production branch**: main（或你的主分支名）
   - **Framework preset**: None
   - **Build command**: 留空
   - **Build output directory**: `public`
6. 点击 "Save and Deploy"

### 步骤 4：配置前端使用后端地址

修改前端代码中的 API 地址，从本地地址改为 Vercel 后端地址。

## 方案二：全栈部署到 Vercel

只需要执行方案一中的步骤 2 即可，Vercel 会同时托管前端和后端。

## 方案三：重写为 Cloudflare Workers（高级）

需要将 server.js 重写为 Cloudflare Workers 格式，并使用 Cloudflare KV 存储数据。

## 注意事项

1. **数据持久化**：由于这个项目使用本地文件存储数据，在无服务器环境（Vercel/Cloudflare）中数据不会持久化。建议：
   - 使用 Cloudflare KV 或 Vercel KV 存储配置
   - 使用 Cloudflare D1 或 Vercel Postgres 存储日志和统计数据

2. **CORS 配置**：如果前后端分开部署，需要在后端配置 CORS

3. **环境变量**：敏感配置应该使用环境变量，而不是硬编码

4. **免费额度**：
   - Cloudflare Pages：无限免费
   - Vercel Hobby：每月 100GB 带宽，100GB-hours 服务器时长
